# Code of Conduct

CairnID project spaces should stay professional, precise, and useful for people reviewing, improving, or operating the software and its documentation.

## Expected Behavior

- Treat other contributors and maintainers with respect.
- Keep technical disagreement focused on evidence, tradeoffs, and project goals.
- Avoid sharing private information, credentials, exploit details, or sensitive operational data in public channels.
- Keep issue and pull request comments relevant to the site, documentation, or confirmed project process.
- Give maintainers enough detail to reproduce site build, documentation, or behavior reports.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing someone else's private information without explicit permission.
- Publicly disclosing unresolved security vulnerabilities in a way that puts users or operators at risk.
- Spam, sustained disruption, or bad-faith issue or pull request activity.
- Pressure to bypass review, ignore security concerns, or merge unverifiable changes.

## Reporting

Use only conduct reporting routes listed in public repository or organization settings/docs.

Do not post private personal data or sensitive conduct reports in public issues. For vulnerabilities in this site repository, follow [SECURITY.md](SECURITY.md) instead of opening a public issue.

Maintainers may remove comments, close issues or pull requests, limit participation, or block accounts when behavior puts the project or community at risk.
